# Mini OJ 实验报告

## 1. 项目概述

项目使用 FastAPI、SQLite 和原生 JavaScript，实现用户认证、题目管理、异步 Python 评测、提交查询、分级日志、审计以及备份恢复。学生可完成题目浏览、代码提交和结果查询；教师可管理题目；管理员可管理用户和创建备份。

## 2. 系统架构

系统分为路由层、权限依赖、业务服务、SQLite 数据层、子进程评测层和浏览器前端。

## 3. 数据设计

主要实体包括用户、题目、测试点、提交、测试点日志、审计日志和备份记录。

## 4. 核心实现

提交接口创建 pending 记录并返回 202，FastAPI 后台任务通过统一状态机将其改为 running，在独立 Python 子进程逐测试点执行。输出按换行符、行尾空白和末尾空行规范化。系统按 SE、TLE、RE、WA、AC 的优先级生成最终结果。非法状态跳转会被拒绝，重启后遗留任务会标记为 failed/SE。学生日志视图裁剪隐藏数据并脱敏路径。SQLite 备份使用在线 backup API 生成，并通过 manifest 和完整性检查保护恢复过程。

## 5. API 说明

接口以 `/api` 为前缀，完整清单可通过 FastAPI `/docs` 查看。

## 6. 测试结果

执行 `pytest` 查看自动化测试结果。当前测试覆盖 AC、WA、RE、TLE、输出规范化、注册登录、角色权限、题目校验、隐藏测试点、提交所有权、状态机、重新评测、完整日志审计、路径脱敏、日志截断、用户审计、备份恢复和损坏备份保护。最终提交前应补充实际测试截图和环境信息。

## 7. 问题与解决过程

1. 自定义 Pydantic 校验错误中包含不可直接 JSON 序列化的异常对象，导致 422 请求出现服务器错误。通过统一使用 `jsonable_encoder` 转换验证错误解决。
2. Windows 下校验损坏 SQLite 备份时，异常路径没有关闭连接，导致备份文件被锁定。通过 `try/finally` 保证连接释放，并增加损坏备份自动化测试解决。

## 8. AI 工具使用说明

项目使用 Codex 辅助整理需求、生成基础代码和测试。提交者需要逐项运行测试、审阅实现并能够解释核心代码。
