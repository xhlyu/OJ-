# Mini OJ 实验报告

## 1. 项目概述

项目使用 FastAPI、SQLite 和原生 JavaScript，实现用户认证、题目管理、异步 Python 评测、提交查询、分级日志、审计以及备份恢复。

## 2. 系统架构

系统分为路由层、权限依赖、业务服务、SQLite 数据层、子进程评测层和浏览器前端。

## 3. 数据设计

主要实体包括用户、题目、测试点、提交、测试点日志、审计日志和备份记录。

## 4. 核心实现

提交接口创建 pending 记录并返回 202，后台 asyncio 任务将其改为 running，在独立 Python 子进程逐测试点执行。输出按换行符、行尾空白和末尾空行规范化。系统按 SE、TLE、RE、WA、AC 的优先级生成最终结果。学生日志视图裁剪隐藏数据并脱敏路径。

## 5. API 说明

接口以 `/api` 为前缀，完整清单可通过 FastAPI `/docs` 查看。

## 6. 测试结果

执行 `pytest` 查看自动化测试结果。最终提交前应补充实际测试截图和环境信息。

## 7. 问题与解决过程

待结合实际调试过程补充至少两个问题。

## 8. AI 工具使用说明

项目使用 Codex 辅助整理需求、生成基础代码和测试。提交者需要逐项运行测试、审阅实现并能够解释核心代码。
